# 青年資源整合 — 編輯個人資料 畫面結構

> **Target:** Expo (React Native, iOS-first)
> **Scope:** 編輯個人資料頁 / EditProfile(從「我的」→ UserCard / 個人資料 進入)
> **使用者:** 王小華 / 國立中山大學 · 資工系大三 · 22 歲
> **對應 parent:** `profile-design.md`

---

## 設計原則(必守,沿襲 profile-design.md)

編輯個人資料 = **表單輸入 + 分區管理 + 即時驗證** 的任務型頁面。

- ✅ 表單欄位依性質**分區歸組**(基本、學業、聯絡、個人偏好),每區一張 SectionCard + caps 小標
- ✅ 同一 SectionCard 內的欄位**共用一張卡**,用 hairline divider 分隔,不再每個欄位獨立一張白卡
- ✅ Label 固定在輸入框**上方**(float label 風格),聚焦時珊瑚色、一般時次要色
- ✅ 頭部保留 Back 與「儲存」按鈕,儲存鈕**僅在表單有變動時**才變成可點(珊瑚實心),否則為淺底停用態
- ✅ 所有可選項(性別、身份類別、目前狀態)用**膠囊單選 chip**,選中為珊瑚實心、未選為灰線框
- ✅ 重要/不可逆動作(變更 Email、手機)以 **Row + chevron** 形式跳到子頁,不直接 inline 編輯
- ❌ 不用 emoji,所有 icon 自繪 SVG
- ❌ 不把每個欄位各自包一張卡(現況問題 — 視覺破碎)
- ❌ Save 按鈕不使用對比過強的紅色,統一走珊瑚主色

## 現況問題診斷(為什麼要重設計)

對照目前截圖:

| 問題 | 現況 | 新設計 |
|---|---|---|
| 卡片氾濫 | 每個欄位都一張白卡,6+ 張卡片堆疊 | 依語意分 4–5 區,每區一張 SectionCard |
| 缺少分區 | 所有欄位扁平列出,無層級 | 用 caps 小標分「基本資訊 / 學業 / 聯絡 / 個人狀態 / 進階」 |
| 無頭像編輯 | 頁面沒有頭像區 | 頂部加 AvatarEditor,可換色 / 首字 |
| Label 在框內 | 點擊後 label 消失易混淆 | Label 固定在框上方,float-style |
| 儲存鈕永遠亮 | 無變動也可點 | 有 dirty 才亮,並加確認回饋 |
| 狀態 chip 樣式弱 | 僅灰框,無選中態 | 選中珊瑚實心、未選灰線框 |
| 缺返回前的 unsaved 警告 | 直接 pop 會遺失輸入 | 有變動時 back 觸發 Alert |

## 關鍵元件內部結構

### NavHeader(浮動膠囊導航列)

```
┌────────────────────────────────────┐
│  [‹]       編輯個人資料      [儲存] │
└────────────────────────────────────┘
```

- **左側 Back**:36×36 白底圓鈕 + 淺陰影,內嵌 chevron-left outline
- **中間標題**:身體大字 semibold,置中
- **右側儲存**:
  - **無變動**:淺珊瑚底 + 珊瑚文字(停用觀感,但仍可點顯「目前沒有變更」Toast)
  - **有變動 (dirty=true)**:珊瑚實心底 + 白字,加入輕微發光
  - 點擊後:loading spinner 取代文字 → 成功後 Toast「已儲存」+ 退回上一頁
- Header 背景:隨 scroll 加入 BlurView 玻璃模糊,scroll 到頂時透明

### AvatarEditor(頭像編輯區)

```
         ┌─────────┐
         │         │
         │    王    │   ← 80×80 漸層圓 + 首字白色 32pt bold
         │       📷 │   ← 右下 24×24 白底圓 + camera icon
         └─────────┘
          變更頭像 ▾
```

- 頭像本體:80×80 圓形,**漸層底(珊瑚→淺橙)+ 姓氏首字**
- 右下角相機徽章:24×24 白圓 + 1pt 珊瑚邊 + camera outline icon
- 點擊頭像或「變更頭像」→ 開 **BottomSheet**,選項:
  - 變更漸層配色(6 組預設)
  - 使用其他字元(全名首字 / 英文名首字)
  - (未來)上傳照片(目前灰色 disabled,加「即將推出」tag)
- 中央置中,上下內邊距 24

### SectionCard(區段卡容器,沿用 profile-design.md)

- 整段包一張白卡:`borderRadius: 18` + 微陰影(shadow opacity 0.04)
- 卡內每列為 FormRow,用內部 hairline divider 分隔(divider 從內邊距 16 開始,不貼左邊緣)
- 卡外上方放 caps 小標(section header),caps 全大寫、次要色、`letter-spacing: 0.8`、左邊距對齊卡片內容(16)
- 卡與卡之間垂直間距 24

### FormRow 變體(統一樣式族)

所有表單列共用 `FormRow` 基礎元件,依 **type** 切換內部:

#### A. TextField(單行文字)

```
┌────────────────────────────────────┐
│ 顯示名稱                            │  ← label(小字次要色,focus 轉珊瑚)
│ 王小華                        [✕]  │  ← 輸入值(身體字),右側清除鈕
│ ────────────────────────────────── │  ← 底線 1pt(focus 時珊瑚 2pt)
└────────────────────────────────────┘
```

- Label 固定置頂,**不**隨輸入浮動消失
- Padding:`V 12 H 16`
- Focus 態:底線變珊瑚 2pt,label 變珊瑚
- Error 態:底線紅色,下方顯錯誤訊息(小字)
- 右側清除 `✕`:僅在有值且 focus 時顯示,12×12

#### B. TextArea(多行文字)

```
┌────────────────────────────────────┐
│ 自我介紹                     28/120 │  ← 右側字數計(接近上限轉珊瑚)
│ 喜歡研究 AI,熱愛產品設計,        │
│ 正在找暑期實習機會。                │
│ ────────────────────────────────── │
└────────────────────────────────────┘
```

- 最少 2 行高度,可隨內容 grow 最多 4 行後開始 scroll
- 字數計算超過 90% 上限時珊瑚色、100% 紅色並禁止繼續輸入

#### C. ChipSelector(膠囊單選 / 多選)

```
┌────────────────────────────────────┐
│ 性別                                │
│ (男)  (女●)  (不透露)              │  ← 選中:珊瑚實心 + 白字
└────────────────────────────────────┘
```

- 每個 chip:`paddingH 14 V 8`、`borderRadius 999`
- 未選:白底 + 1pt 淺灰邊 + 次要色文字
- 選中:珊瑚實心 + 白字 + 無邊
- 點擊有 scale 0.96 feedback
- 多選型(興趣標籤)選中後右側加小 `✕` icon,可一鍵移除

#### D. SliderRow(滑桿)

```
┌────────────────────────────────────┐
│ 年齡                          22 歲 │  ← 右側顯示目前值
│ ●━━━━━━━━○━━━━━━━━━━━━━━━━━━━     │
│ 15                              35  │  ← 兩端小字區間
└────────────────────────────────────┘
```

- Track:已滑過段珊瑚實色,未滑過段淺灰
- Thumb:珊瑚實心圓 + 白描邊 + 微陰影
- 拖曳時 thumb 放大 1.15x,目前值用氣泡浮在 thumb 上方
- 左右兩端顯示區間小字

#### E. NavRow(跳轉列,不 inline 編輯)

```
┌────────────────────────────────────┐
│ ✉  Email               已驗證   ›  │
│    test@gmail.com                   │
└────────────────────────────────────┘
```

- 結構同 profile-design.md 的 SettingRow
- 用於需跳到子頁編輯的欄位(Email、手機、學校、地區、密碼)
- 主標題在上、**目前值**在下(身體字次要色)
- 右側 value slot:
  - `已驗證`(綠色小字)
  - `前往驗證`(珊瑚色,未驗證時)
  - 空白(一般跳轉)
- 最右 chevron `›` 淺灰

### UnsavedChangesGuard(未儲存攔截)

- 使用 `useEffect` 監聽 `dirty` state
- 當 `dirty=true` 且使用者按 Back / 手勢返回:
  - 跳 Alert:「有尚未儲存的變更,確定要離開嗎?」
  - 選項:`繼續編輯`(預設) / `放棄變更`(紅字) / `儲存並離開`(珊瑚)

| Type | 欄位 | 右側狀態 |

## 互動細節

- **輸入變動**:任一欄位 onChange 立即標記 `dirty=true`,儲存鈕由淺轉實心
- **欄位驗證**:
  - 即時驗證:顯示名稱長度、Email 格式、手機格式
  - 失焦驗證:錯誤以紅色底線 + 底下小字錯誤訊息
  - 送出前總驗證:若有 error,自動 scroll 到首個錯誤欄位並 focus
- **儲存流程**:
  1. 點擊儲存 → 按鈕替換為 spinner
  2. 成功 → Toast「已儲存」+ `dirty=false` + 退回上一頁
  3. 失敗 → Toast 紅底錯誤訊息 + 保持在頁
- **返回攔截**:見 UnsavedChangesGuard
- **鍵盤處理**:
  - `KeyboardAvoidingView` 包整頁
  - 點擊輸入框外任意處 → 收鍵盤
  - TextArea focus 時自動 scroll 到可見區
- **AvatarEditor 點擊**:開 BottomSheet,含 6 組漸層色卡(珊瑚/琥珀/薰衣/湖綠/天藍/玫瑰)
- **ChipSelector 多選(興趣標籤)**:
  - 點擊 `+ 新增` → BottomSheet 分類展示所有可選標籤,點選即加入
  - 點擊已選 chip 右側 `✕` → 直接移除
  - 長按已選 chip → 進入重排序模式

## 色彩 / 字體 tokens(沿用 profile-design.md)

- **珊瑚主色** `#FF6B6B` 系:用於 Save 實心態、focus 底線、選中 chip、slider 主色、小勾與主要強調
- **珊瑚淺底** `#FFE8E8`:Save 無變動態、chip 悬停
- **綠驗證色** `#10B981`:已驗證狀態文字
- **紅警示色** `#EF4444`:停用帳號、刪除、錯誤訊息
- **次要文字色** `#9CA3AF`:label、副標、右側 value 純文字
- **背景** `#F7F7F8`:頁面底色(與 SectionCard 白色形成層次)
- **字體**:
  - Caps 小標:11pt / semibold / letter-spacing 0.8 / 次要色 / 全大寫
  - 欄位 label:13pt / regular / 次要色
  - 輸入值:16pt / regular / 主文字色
  - 儲存按鈕:15pt / semibold

## 無障礙

- 所有 FormRow 綁定 `accessibilityLabel`,讀屏友善
- Slider 支援 `accessibilityIncrements`,每次 ±1 歲
- Chip 選中狀態用 `accessibilityState: { selected: true }`
- 色彩對比全部符合 WCAG AA(珊瑚實心 + 白字對比 ≥ 4.5)
- 所有可點區域 hitSlop 至少 44×44
